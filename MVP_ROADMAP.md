# SellerPilot AI — комплексный план запуска

## Позиционирование

**SellerPilot AI — AI-директор для продавцов Ozon и Wildberries.**

Не просто аналитика и не тяжёлая ERP. Сервис каждый день говорит селлеру:

- что пополнить;
- какую карточку исправить;
- где теряются продажи;
- какие SKU перенести на второй маркетплейс;
- где проверить цену, рекламу или маржу;
- что сделать сегодня в первую очередь.

## Главный оффер

> Подключите Ozon и Wildberries — получите ежедневный AI-план действий для роста продаж и контроля остатков.

## Почему это можно продать, несмотря на аналоги

На рынке уже есть SelSup, Точка, MPStats, Uniseller, EcomCom, МойСклад-интеграции и другие. Поэтому нельзя идти как «ещё один единый кабинет».

Нужно идти как:

> **AI-операционный директор для малого селлера.**

Отличие:

- меньше таблиц;
- меньше сложных отчётов;
- больше готовых действий;
- быстрый onboarding;
- фокус только на Ozon + WB;
- понятная цена для малого бизнеса.

## MVP 1 — уже заложено в проекте

Файлы:

- `index.html` — лендинг продаж;
- `app.html` — рабочий кабинет;
- `server.js` — backend-прокси для реальных Ozon/WB API;
- `README.md` — запуск и подключение;
- `MVP_ROADMAP.md` — этот план.

Функции MVP:

1. Подключение WB API token.
2. Подключение Ozon Client-Id + Api-Key.
3. Синхронизация товаров.
4. Получение остатков.
5. Получение цен, если права токена позволяют.
6. Единый список товаров.
7. Единая страница остатков.
8. Обновление остатков обратно в маркетплейсы.
9. Rule-based AI-рекомендации.
10. Опциональное подключение LLM через `AI_API_KEY`.

- Логистика.
- Хранение.
- Штрафы.
- Чистая прибыль по SKU.

### 3. Модуль карточек

- Проверка названий.
- Проверка описаний.
- Проверка характеристик.
- Проверка фото.
- SEO-рекомендации.
- Генерация улучшенного описания.

### 4. Модуль цен

- Массовое изменение цен.
- Минимальная маржа.

- Загрузка рекламных кампаний.

### 6. Модуль отзывов и вопросов

- Единая лента отзывов Ozon/WB.
- AI-ответы в стиле бренда.
- Алерты по негативу.
- Выявление причин возвратов.

## Формат AI-задачи

Каждая рекомендация должна быть не просто текстом, а задачей:

```json
{
  "priority": "high",
  "marketplace": "wb",
  "sku": "WB-1001",
  "title": "Пополнить остаток",
  "reason": "Остаток 0, товар был активен и может терять продажи",
  "action": "Поставить минимум 42 шт.",
  "expectedImpact": "Вернуть товар в продажу",
  "confidence": 0.82
}
```

## MVP-метрики

- Количество подключённых магазинов.
- Количество синхронизированных SKU.
- Количество созданных AI-задач.
- Сколько задач пользователь применил.
- Сколько out-of-stock исправлено.
- Retention 7 дней.
- Сколько пользователей подключили оба маркетплейса.

## Первые клиенты

Идеальные первые клиенты:

- 100–3000 SKU;
- продают одновременно на Ozon и WB;
- устали от Excel;
- не готовы платить за тяжёлую ERP;
- хотят понятный список действий, а не просто графики.

## Короткий питч

> SellerPilot AI подключается к Ozon и Wildberries, анализирует товары, остатки, цены и карточки, а затем каждый день выдаёт селлеру конкретный план действий: что пополнить, что исправить и где теряются деньги.
